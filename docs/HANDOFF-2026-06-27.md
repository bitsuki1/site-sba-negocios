## 1. O que é / como rodar
- Stack: **Vite + React + TS + Tailwind + shadcn/ui** (Lovable-ready). `npm install && npm run dev` (porta 8080; em
  ambiente sem IPv6, use `--host 127.0.0.1`). `npm run build` gera `dist/`. `tsc --noEmit` e `vite build` passam limpos.

## 2. O que está PRONTO
- **10 produtos, cada um com landing própria.** Páginas ricas: **Recuperação Tributária** (Tema 1130, com FAQ),
  **Resíduos** (`/residuos`), **Resíduos Hospitalares** (`/residuos-hospitalares`), **Estabilizador de Solo**
  (`/estabilizador-de-solo`, com ConAid CBR Plus + Dynacal). Demais via gerador de landing data-driven
  (`/solucoes/:slug`): RSU municipal, grandes geradores, fotovoltaica, iluminação pública, reflorestamento & carbono,
  saneamento/água, eventos/cultura.
- **Arquitetura híbrida:** Home com 2 portas (Setor Público / Setor Privado) + hubs por setor + página Soluções por tema.
- **Logo da SBA** no header e rodapé. **Imagens reais** (dos materiais dos parceiros CSTR/ConAid) nas páginas de resíduos,
  RSS e estabilizador.
- **Leads no Supabase** via **Edge Function `submit-lead`** (projeto `gestao-integrada-dados`): grava em `sba_leads` com
  service role; o site NUNCA escreve direto; `anon` não LÊ a tabela (RLS). Chave **publishable** no front é pública por
  design. **`service_role` nunca vai pro front.** Fallback por e-mail (mailto) preservado.
- **Marca SBA** em todo lugar (0 marca proibida). Honestidade tributária: "piso estimado / no êxito"; nada de
  "garantido/comprovado". Área B: "valores sob consulta", sem número inventado.
